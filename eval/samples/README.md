# Sample Organization

Checked-in PDF fixtures remain under `fixtures/`, but issue `#42` evals prioritize public knowledge and guidance PDFs cached under `eval/public-samples/`.

Eval sample policy:

- `smoke`: smallest deterministic public dense docs, no provider dependency
- `core`: representative knowledge/guidance PDFs with stable expectations
- `stress`: long-page, multi-page, or budget-sensitive guidance docs
- `known-bad`: intentionally retained regressions or unsupported patterns that should not silently disappear

Placement rules:

- public official PDFs should be fetched by `eval/fetch-public-samples.mjs`
- deterministic synthetic cases should be generated by `eval/run-local.mjs` only when public samples cannot cover a behavior
- real PDFs that must be versioned stay in `fixtures/`
- each sample gets one stable `caseId`
- `caseId` must describe the dominant failure mode or capability under test
- expected outputs belong in manifests, not ad hoc notes

Naming:

- suite ids: `smoke`, `core`, `stress`, `known-bad`
- sample ids: `<modality>-<behavior>-<shape>`
- config ids: `<modality>-<model-or-mode>-<budget-or-prompt>`
